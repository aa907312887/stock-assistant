# 数据模型说明：破 60 日均线买入法

**关联规格**：[spec.md](./spec.md)  
**结论**：**不新增**业务表；复用现有 **`stock_daily_bar`**、**`stock_basic`**、回测结果表 **`backtest_task` / `backtest_trade`**（或项目当前等价命名）。

---

## 1. 依赖的已有表与字段

### 1.1 `stock_daily_bar`（或项目实际日表名）

| 用途 | 字段（逻辑名） | 说明 |
|------|----------------|------|
| 时间轴 | `stock_code`、`trade_date` | 按标的 + 日期升序组 K 线序列 |
| 突破/均线下判定 | `close`、`ma60` | 均须**非空**才参与当日判定；`ma60` 为库内已预计算 60 日收盘均线 |
| 买入价 | `open` | **信号日下一行**的开盘价；缺则**不成交** |
| 持仓期卖出监测 | `close` | 自**买入行下一根**起按**收盘价**与 `buy_price` 算收益，**先止损后止盈** 8%（见 [research.md](./research.md)） |

**停牌与缺行**：无独立字段；**无行则该日不存在于序列中**，不额外「补停牌日」。

### 1.2 `stock_basic`

| 用途 | 字段 | 说明 |
|------|------|------|
| 名称展示 | `code`、`name` | 回测明细中 `stock_name`；**ST/*ST** 与 `ma60_slope_buy` 同样**剔除**（实现时复用同模式） |

### 1.3 回测任务与交易明细

| 实体 | 说明 |
|------|------|
| 回测任务 | 与现网一致：绑定 `strategy_id=ma60_five_day_break`、区间、资金参数、状态等 |
| 交易明细 | `trigger_date` = 信号日 D；`buy_date` = D+1 交易日；`buy_price` = 次日开盘；`sell_date`/`sell_price` = 按收盘价触发的卖出日/价；`extra_json` 可存 `exit_reason`、各日 close/ma60 快照键名（与实现约定一致即可） |

---

## 2. 校验与不变量

- 信号日下标 `i` 需满足 `i >= 5` 且存在 `i+1`（买入行）；且 `D−5…D−1` 与 `D` 的 `close`、`ma60` 均可比。
- 全市场扫描时与现有策略相同：按 `stock_code` 分组、组内时间升序；单标的**未平仓**前是否扫描后续信号由**回测引擎**与 `ma60_slope_buy` **相同**的 `last_block` 语义处理（见实现计划正文）。

---

## 3. 与指数/回测 `symbols` 的关系

本策略默认**全市场个股**；若未来与 `ma_golden_cross` 一样支持 `symbols` 限定，属**扩展项**，不在 P1 规格内强制；数据模型**仍**仅依赖 `stock_daily_bar`。
