# 调研结论：破 60 日均线买入法（历史回测）

**日期**：2026-04-18  
**关联规格**：[spec.md](./spec.md)  
**本文件作用**：冻结 Phase 1 实现前的**业务/技术**决策，供 `plan.md` 与代码对齐。

---

## 1. 「不考虑停牌」如何落地

| 决策 | **以 `stock_daily_bar` 中已落库、按 `trade_date` 升序排列的连续 K 线行**作为该标的的「交易日」序列；**不**单独读取停牌标记、**不**向交易所日历借「应交易日」补行。 |
| 理由 | 与仓库内现有日线回测（如 `ma60_slope_buy`）一致；用户明确**不考虑停牌**即不在策略内增加停牌专用分支。若某日无行（含停牌、未同步），则**不**在当日生成价格，前 5 日 + 信号日 + 买入日**必须**均对应**数据行**；缺行则该窗口**不**构成有效信号。 |
| 备选 | 用 `trade_cal` 对齐自然周：实现成本高，且与「不考虑停牌」、现有引擎习惯不一致，**不采用**。 |

**与规格 FR-003/FR-004 的对应关系**：在实现上，**D−5…D−1 与 D** 均取 `bars_list` 中**下标** `i-5…i-1` 与 `i` 的**连续**五根 + 信号根；即**五根在均线下方 + 一根突破**均在**同一条时间序列**上，天然为「有 K 的交易日」。

## 2. 止盈/止损「同日双触」处理顺序

| 决策 | 与 `ma60_slope_buy` 中 `simulate_exit_close_only` **一致**：自 **买入日下一根 K** 起按**收盘价**监测；**先判止损**（`close ≤ 买入×(1−0.08)`），再判止盈（`close ≥ 买入×(1+0.08)`）。 |
| 理由 | 与现有 8% 止损语义一致、可复用或复制同一循环结构；极端情况下**唯一**、**可复现**。 |
| 备选 | 先止盈：在剧烈向下缺口时与「保守止损」直觉冲突，**不采用**除非产品统一改口径。 |

---

## 3. 与 `022-60日均线回测`（`ma60_slope_buy`）的区分

| 决策 | **新**策略使用**新** `strategy_id` 与**新**模块文件，**不**在 `ma60_slope_buy` 上开参数开关混用两种形态。 |
| 理由 | 买入条件、止盈比例（8% vs 15%）均不同，分策略便于维护、回测对比与注册表展示。 |

---

## 4. 展示名与 `strategy_id` 建议

| 决策 | `strategy_id`：固定为 `ma60_five_day_break`（蛇形、与仓库一致）。**展示名**：`破60日均线买入法`（与规格 FR-001 一致，可在 `describe()` 中微调短描述）。 |
| 理由 | 与 `ma60_slope_buy`（「60日均线买入法」）名称区分，避免用户混选。 |

---

## 5. 是否需要新定时任务

| 决策 | **本功能不新增**定时任务；与同类内置策略一样，**可选**由既有「按策略 id 跑选股/筛选」的调度在后续接入时增加一行配置（非本规格必选）。 |
| 理由 | 规格 P1 为**历史回测**闭环；定时选股与 `021` 等一致，可二期再对 `execute()` 与 scheduler 对表。 |
