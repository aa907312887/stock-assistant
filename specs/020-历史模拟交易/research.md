# 技术调研：历史模拟交易

**日期**: 2026-04-09 | **规格**: `./spec.md`

## 1. 与现有「历史模拟」功能的关系

**决策**：本功能（交互式手动模拟交易）与现有 `simulation_task` / `simulation_trade` 表及 `/api/simulation` 路由**完全独立**，新建独立的数据表和 API 路由前缀 `/api/paper-trading`。

**理由**：现有 simulation 是「策略自动回测」——系统自动遍历历史数据、批量产生交易记录，无用户交互；本功能是「用户手动逐日操作」——用户主动选股、手动输入价格、逐日推进，两者数据模型、业务逻辑、交互模式完全不同，共用表会导致字段语义混乱。

**备选**：复用现有表并加 `type` 字段区分 → 否决，因为字段差异太大（现有表无持仓、无资金、无 FIFO 批次概念）。

## 2. K 线图表库选型

**决策**：使用 **ECharts**（项目已有依赖）实现 K 线图、MACD、均线、成交量图。

**理由**：
- 项目前端已使用 ECharts（其他页面有图表），无需引入新依赖。
- ECharts 内置 `candlestick`（K 线）、`bar`（成交量）、`line`（均线/MACD）图表类型，支持多图联动（dataZoom 同步缩放）。
- 支持 tooltip 悬停显示 OHLCV 数据。

**备选**：TradingView Lightweight Charts → 功能更专业，但需引入新依赖，且与项目现有技术栈不统一。

---

## 3. 持仓批次与 FIFO 卖出实现

**决策**：后端数据库按批次存储持仓（每次买入一条记录），卖出时后端执行 FIFO 逻辑，前端展示时按股票代码聚合显示加权均价。

**理由**：
- FIFO 逻辑在后端执行，保证数据一致性，前端无需关心批次细节。
- 前端持仓列表按股票聚合（加权均价 + 总数量），符合用户直觉。
- 底层批次记录保留，支持精确的 T+1 判断（每批次独立判断是否可卖）。

**FIFO 卖出算法**：
```
卖出 N 股时：
1. 查询该会话该股票所有持仓批次，按 buy_date ASC、id ASC 排序
2. 过滤掉 buy_date == 当前模拟日期 的批次（T+1 限制）
3. 从最早批次开始扣减，直到扣完 N 股
4. 若可卖总量 < N，拒绝交易并提示
```

## 4. 交易日历实现

**决策**：不维护独立交易日历表，通过查询 `stock_daily_bar` 中已有数据的 `trade_date` 集合推断交易日。

**理由**：`stock_daily_bar` 已覆盖全 A 股历史日线数据，有数据的日期即为交易日，无需额外维护。

**实现**：后端提供 `GET /api/paper-trading/trading-dates?start=&end=` 接口，返回指定范围内的交易日列表（从 `stock_daily_bar` 中 `SELECT DISTINCT trade_date` 获取）。

## 5. 涨跌停价格计算

**决策**：涨跌停价格 = 前收盘价 × (1 ± 10%)，取两位小数（四舍五入）。数据来源：`stock_daily_bar.prev_close`。

**理由**：`stock_daily_bar` 已有 `prev_close` 字段，可直接计算，无需额外查询。

**简化假设**：统一按 ±10% 计算，不区分 ST 股（±5%）、科创板首日等特殊情况。

---

## 6. 手续费计算

**决策**：
- 买入手续费 = max(成交金额 × 0.03%, 5 元)
- 卖出手续费 = max(成交金额 × 0.13%, 5 元)（含 0.1% 印花税）
- 成交金额 = 价格 × 数量

**理由**：符合 A 股主流券商费率标准，与 spec 约定一致。

---

## 7. 股票推荐与筛选的数据来源

**决策**：随机推荐和自定义筛选均基于当日 `stock_daily_bar` 数据，停牌判断依据为当日是否有数据（无数据 = 停牌）。

**均线金叉判断**：
- MA5 上穿 MA10：当日 `ma5 > ma10` 且前一日 `ma5 <= ma10`
- MA5 上穿 MA20：当日 `ma5 > ma20` 且前一日 `ma5 <= ma20`

**MACD 金叉判断**：当日 `macd_dif > macd_dea` 且前一日 `macd_dif <= macd_dea`

**理由**：`stock_daily_bar` 已预计算 MA5/MA10/MA20/MA60 和 MACD 指标，无需实时计算。

---

## 8. 图表数据接口设计

**决策**：图表数据通过独立接口 `GET /api/paper-trading/chart-data` 提供，参数包含股票代码、截止日期、周期（日/周/月）、数据条数（默认 300）。

**理由**：
- 截止日期参数确保不泄露未来数据（`trade_date <= current_sim_date`）。
- 日/周/月分别查询 `stock_daily_bar` / `stock_weekly_bar` / `stock_monthly_bar`。
- 返回 OHLCV + MA5/MA10/MA20/MA60 + MACD(DIF/DEA/HIST)，前端直接渲染。

---

## 9. 会话持久化

**决策**：模拟交易会话存储在数据库（MySQL），支持多会话并存，用户可随时保存并恢复。

**理由**：与项目现有数据持久化方案一致（SQLAlchemy + MySQL），无需引入额外存储。

**会话标识**：`session_id` 使用 UUID，格式 `pt-{uuid8}`（pt = paper trading）。
