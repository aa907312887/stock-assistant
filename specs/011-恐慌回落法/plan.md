# 实现计划：恐慌回落法（历史回测）

**分支**：`main` | **日期**：2026-03-27 | **规格**：`./spec.md`  
**状态**：已实现（策略 + 回测结果页筛选/分年/最佳组合等与 spec 同步）  
**输入**：功能规格来自 `specs/011-恐慌回落法/spec.md`

## 概要

本期交付一个可回测的策略：**恐慌回落法**。系统能够在指定股票（或股票池）与区间内逐日扫描信号，按“触发日收盘买入、次日收盘卖出”的口径生成逐笔交易，并输出汇总指标用于历史验证与参数对比。

本功能优先复用现有“智能回测”基础设施（回测任务、交易明细表、策略框架与接口），仅新增：

- 一个新策略实现（恐慌回落法）及其参数
- 策略在回测引擎中的注册与可选择
- 指标计算/展示复用既有能力（如已有）

## 技术背景

- **后端**：FastAPI + SQLAlchemy + MySQL（项目既有）
- **数据**：日线 OHLCV（已存在的行情表/服务）
- **回测方式**：日线逐日扫描，交易价使用收盘价近似（见规格）

`constitution.md` 目前为占位模板，缺少可执行门禁条款。本计划遵循项目既有工作方式：Spec 驱动、文档中文、尽量复用现有回测通用框架，避免超范围扩展（不引入分时回测、不引入复杂风控）。

## 关键设计详述

### 设计原则

- **口径第一**：严格按 `specs/011-恐慌回落法/spec.md` 的信号与成交价口径实现，确保回测结果可复现。
- **复用优先**：若 `specs/010-智能回测/plan.md` 中的回测引擎与数据表已落地，则本策略以“新增策略模块”的方式接入；不重复造轮子。
- **参数可对比**：策略参数需可配置并落库到回测任务（或能在结果中追溯）。

### 数据流

```
用户选择策略=恐慌回落法 + 时间范围 + 股票池
  → 创建回测任务
  → 回测引擎逐股票逐日扫描信号
  → 命中信号则生成 1 笔交易：
       买入价=触发日收盘价
       卖出价=次日收盘价
  → 落库交易明细 + 汇总指标
  → 前端展示交易列表与指标
```

### 策略接口与注册

- **策略标识**：`panic_pullback`（对外展示名“恐慌回落法”）
- **策略输入**：
  - 日线序列（至少覆盖回测区间 + 指标窗口）
  - 策略参数（均线周期、阈值、成交量系数 k）
- **策略输出**：
  - 逐笔交易（触发日、买入/卖出日期、买卖价、收益率）
  - 汇总指标由回测引擎/报告模块统一计算（复用）

### 关键算法细节（按规格直接落地）

- **均线**：简单移动平均（SMA），基于收盘价
- **空头排列**：`MA5 < MA10 < MA20`
- **过去 5 天跌 4 天**：统计 `Close_i < Close_{i-1}` 的天数
- **低开 ≥3%**：`Open_t <= Close_{t-1} * 0.97`
- **整体跌 ≥7%**：`Close_t <= Close_{t-1} * 0.93`
- **成交量显著放大（B 方案）**：
  - `Vol_t > max(Vol_{t-1..t-5})`
  - `Vol_t >= k * avg(Vol_{t-1..t-5})`，默认 `k=1.5`
- **交易**：
  - 买入：`t` 收盘价
  - 卖出：`t+1` 收盘价
- **不可成交/数据不足**：按规格“边界与口径”处理并披露占比

### 接口与前端影响

- 若项目已存在通用回测接口（见 `specs/010-智能回测/contracts/backtest-api.md`），本期只需确保策略列表包含“恐慌回落法”，以及回测任务参数中能携带并展示该策略参数。
- 若尚未实现策略列表接口，则在回测运行接口中通过 `strategy_key` 选择策略，并在前端下拉框展示“恐慌回落法”。

## 风险与回退

- **风险**：行情数据缺失/停牌导致交易无法按“收盘价”口径成交。
  - **回退**：标记不可成交并单独统计；不强行用替代成交价以免口径漂移。
- **风险**：不同数据源对成交量口径不一致（手/股、复权等）。
  - **回退**：在实现中统一使用数据库/服务既有的成交量字段，并在 quickstart 写清口径。
