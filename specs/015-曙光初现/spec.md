# 功能规格说明：曙光初现（内置策略）

**功能分支**: `main`  
**创建日期**: 2026-03-29（修订：站上 MA5 买入；阳线成交量≥前7日均量×1.5；止损成交价固定买入价×0.94；+10% 后破 MA5 按收盘卖）  
**状态**: 已实现  
**输入**：信号阳线日 T **之前连续 7 个交易日**（T-1…T-7，不含 T）中至少 **5** 天为**阴线**；且 **T-7 收盘至 T-1 收盘**的累计涨跌幅 **≤ -10%**；**跌势**中阳线日须 MA5 小于 MA10 小于 MA20 且阳线收盘仍低于 MA20；前一日阴线且较前一日收盘至少跌 3%；当日阳线且**阳线实体涨幅（相对开盘）≥ 3%**；**T 日成交量 ≥ 前 7 个交易日成交量算术平均值 × 1.5**（比该均值高出二分之一），且 T 与前 7 日 `volume` 均有效；阳线收盘不高于 `cum_hist_high` 的 **50%**；自阳线日起首次 **close > MA5** 买入；持仓后以**收盘价**判定是否触发止损线，触发后**卖出价=买入价×0.94**（固定 −6%，模拟条件单）；曾达收盘 **+10%** 后改按 **跌破 MA5** 的**收盘价**离场。

## 功能需求

### 股票范围

- 扫描 A 股日线；剔除 `ST`、`*ST`。

### 形态（信号阳线日 T）

1. **前期弱势**：在 **T-1 … T-7** 共 7 个交易日中，满足阴线 `close < open` 的天数 **≥ 5**（任一日缺少开/收盘则该标的不触发）。  
2. **前期累计跌幅**：`(close_{T-1} / close_{T-7} - 1) ≤ -10%`（以 T-7、T-1 两日收盘价衡量这 7 日段整体跌幅至少 10%）。  
3. **跌势结构**：`MA5_T < MA10_T < MA20_T`，且 **`close_T < MA20_T`**（阳线当日仍处在长均线之下，排除已呈多头排布、不具备「跌势末端反转」语境的样本）。  
4. T-1 阴线：`close_{T-1} < open_{T-1}`  
5. T-1 相对 T-2 跌幅至少 3%：`(close_{T-1}/close_{T-2} - 1) ≤ -3%`  
6. T 阳线：`close_T > open_T`，且阳线**实体涨幅**（相对当日开盘）**≥ 3%**：`(close_T - open_T) / open_T ≥ 3%`。  
7. T ∈ `[start_date, end_date]`  
8. **相对历史高位**：`close_T ≤ 0.5 × cum_hist_high_T`（`cum_hist_high_T` 为 T 当日行字段）。  
9. **阳线放量**：`volume_T ≥ 1.5 × mean(volume_{T-7}, …, volume_{T-1})`（前 7 日为 T 之前连续 7 个交易日；均值为算术平均）；T 与上述 7 日须均有有效 `volume`。

### 交易规则

- **触发日**：信号阳线日 **T**（形态成立日）；回测交易明细中 **`trigger_date`** 字段记录该日（与 **买入日**可能不同：买入为自 T 日起首次 `close > MA5` 的日期）。  
- **买入**：自 T 日（含）起，**首次**满足 `close > MA5`，以该日**收盘价**买入。  
- **卖出**：自买入日**次日**起（仅考虑到 `end_date`），逐日用**收盘价**判定是否触发规则；顺序如下：  
  1. **止损**：若 `close ≤ 买入价 × (1 − 6%)`，则记为触发止损（`exit_reason`: `stop_loss_6pct`），**卖出价固定为 `买入价 × 0.94`**（收益率 −6%），**不使用**触发日收盘价作为卖出价（与条件单/限价 −6% 成交一致）；`extra` 可含 `trigger_day_close` 记录触发日收盘价供对照。  
  2. 否则若此前已有某日收盘 `≥ 买入价 × (1 + 10%)`，且当日 `close < MA5`，则以**当日收盘价**卖出（`break_ma5_after_arm`）。  
  3. 同一交易日内先判止损，再更新「是否已达 +10%」，再判 MA5；**止损优先**。  
  4. **未达 +10% 前**不因跌破 MA20 单独卖出。  
- **未平仓**：`end_date` 前已买入但未触发上述卖出 → `unclosed`。  
- **同一标的**：出现未平仓后不再扫描该标的后续形态。

### 回测交易明细（通用）

- 表 `backtest_trade` 含可选列 **`trigger_date`**：任意策略可将形态/信号触发日写入该列；未使用则为 `NULL`。曙光初现写入阳线日 T。

### API

- 选股执行：返回 `as_of_date` **当日发生买入**的候选。  
- 回测：`strategy_id` = `shu_guang_chu_xian`；单仓仿真 `allow_rebuy_same_day_as_prior_sell = false`。

## 边界与数据

- `open`、`close`、`volume`、`ma5`、`ma10`、`ma20`、`cum_hist_high` 缺失则跳过相应判定（阳线放量及前 7 日任一日缺量则该样本不成立）。  
- 库表须含 `cum_hist_high` 并已回填（见 `backend/scripts/add_stock_daily_bar_cum_hist.sql` 与 `recompute_hist_extrema_full`）。  
- 若库表尚无 `trigger_date` 列，须执行 `backend/scripts/add_backtest_trade_trigger_date.sql`。
