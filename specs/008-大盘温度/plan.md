# 实现计划：大盘温度

**分支**: `[008-大盘温度]` | **日期**: 2026-03-24 | **规格**: `/specs/008-大盘温度/spec.md`  
**输入**: 功能规格来自 `/specs/008-大盘温度/spec.md`
**计算细则**: `/specs/008-大盘温度/formula.md`（v1.0.0）

**说明**: 本方案围绕“首页右侧嵌入大盘温度模块”落地，覆盖数据拉取、日频计算、入库、接口输出、前端展示与说明交互。

## 概要

本功能在不新增独立页面的前提下，在首页“我的投资逻辑”右侧新增“大盘温度”模块。后端基于上证指数、深证成指、中证500、中证1000四个指数，按交易日计算近三年温度序列，并将总分、三因子分项、档位、策略提示入库。前端拉取最新温度和趋势数据，展示5档可视化状态、策略悬浮提示和“?”口径说明弹层。方案同时支持手动重算与每日自动更新，保证后续可做不同版本温度与策略胜率回测。

## 技术背景

- **语言/版本**: Python 3.x（后端）、TypeScript + Vue 3（前端）
- **主要依赖**: FastAPI、SQLAlchemy、APScheduler（定时任务）、前端现有 UI 组件库
- **存储**: MySQL（温度结果表、分项表、规则表）
- **测试**: pytest（后端）、前端组件/集成测试（沿用仓库既有方式）
- **目标平台**: Linux 服务器 + 现代浏览器
- **项目类型**: Web 应用（前后端分离）
- **性能目标**: 首页温度模块接口 P95 < 300ms；三年历史重算单次 < 5 分钟
- **约束**: 外部行情数据存在限频与偶发缺失；同版本公式固定不可漂移
- **规模/范围**: 约 750~800 个交易日日频结果 + 4 指数分项明细

## 关键设计详述

### 数据流与接口职责

1. 数据采集层（后端脚本/服务）从 Tushare 指数日线接口拉取四个指数近四年数据（四年用于预热窗口，入库三年结果）。
2. 计算层按交易日执行三因子计算（趋势40%、量能30%、风险30%），得到 `temperature_score`（0-100）及5档分级。
3. 持久化层写入三类数据：指数日级行情表 `market_index_daily_quote`、每日总结果、每日分项结果（含公式版本）。
4. API 层对前端提供：
   - 首页卡片接口：最新温度、档位、更新时间、策略提示、5档说明。
   - 趋势接口：最近20交易日温度曲线和升温/降温标记。
   - 口径说明接口：公式版本、因子定义、权重与分级解释。
5. 前端首页在“我的投资逻辑”右侧渲染温度模块，不新增路由页面；悬浮显示策略提示，“?”按钮点击弹出口径说明。

接口职责划分：
- **后端负责**：数据拉取、计算、入库、规则版本管理、接口输出与异常兜底。
- **前端负责**：模块渲染、5档视觉表达、悬浮提示、口径弹层、移动端轻触降级交互。

### 定时任务与部署设计

- **使用的组件**: APScheduler，放在 `backend/app/core/scheduler.py`（若仓库已有调度器则并入现有文件）。
- **注册方式**: 在 FastAPI 生命周期启动阶段注册任务（应用启动时调用 `scheduler.add_job`）。
- **调度策略**: 每个交易日 17:10 执行日更任务（避开收盘后数据延迟），cron：`10 17 * * 1-5`。
- **部署时是否执行一次**: 是。应用启动后延迟 30 秒触发一次“缺口补齐任务”，仅补最近 5 个交易日缺失数据。
- **手动触发方式**:
  - [x] HTTP 接口：`POST /api/admin/market-temperature/rebuild`，鉴权方式：管理员 token。
  - [x] 管理命令：`python -m backend.app.scripts.fill_market_temperature --years 3`
  - [ ] 脚本：不单独新增壳脚本。
- **失败与重试**: 单次任务失败后重试 2 次（间隔 60 秒）；连续失败写 error 日志并记录任务状态为 failed。
- **日志与可观测**: 记录开始/结束、耗时、处理交易日数量、成功条数、失败条数；异常包含指数代码与日期上下文。

### 其他关键设计

- 公式版本管理：新增 `formula_version` 字段，默认 `v1.0.0`；后续调权重需升级版本号，历史数据不可覆盖。
- 指数行情命名：采用 `market_index_daily_quote`，强调“交易日行情数据”语义，避免 `snapshot` 命名歧义。
- 异常兜底：当日计算失败时，首页接口返回最近有效交易日结果，并附 `data_status=stale`。
- 5档规则固定：极冷/偏冷/中性/偏热/过热，默认区间 0-20/21-40/41-60/61-80/81-100。
- 说明文案配置：策略提示与“?”口径说明从后端配置表读取，便于非代码调整。
