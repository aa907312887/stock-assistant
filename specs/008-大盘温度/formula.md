# 大盘温度计算公式（v1.0.0）

## 1. 目的与范围

本文定义“大盘温度”可直接实现的计算细则，适用于：

- 首页大盘温度模块展示
- 近三年日频结果入库
- 后续不同公式版本回测对比

---

## 2. 输入数据与指数池

## 2.1 指数池（固定）

- 上证指数：`000001.SH`
- 深证成指：`399001.SZ`
- 中证500：`399905.SZ`
- 中证1000：`399852.SZ`

默认指数权重均为 25%。

## 2.2 行情字段（交易日级）

来自 `market_index_daily_quote`：

- `trade_date`
- `open`
- `high`
- `low`
- `close`
- `vol`
- `amount`

## 2.3 计算窗口

- MA20：20 个交易日
- MA60：60 个交易日
- 风险波动窗口：20 个交易日
- 每日增量重算回看窗口：最近 80 个交易日（覆盖指标滚动依赖）

---

## 3. 三因子计算定义

所有因子先计算“原始值”，再映射到 0~100 分，最后汇总。

## 3.0 实现口径（均线、波动率、分位数）

以下与 `formula_engine.py` 保持一致，便于文档与代码对照：

- **滚动均线 MA20 / MA60**：对每个指数按交易日升序。若截止当日 historic 长度不足 20（或 60），则用**已存在的全部收盘**求算术平均；长度足够时取最近 20（或 60）个收盘求算术平均。
- **20 日波动率 vol20**：取该指数**已产生的日收益序列**的最近至多 20 项 \(r_1,\ldots,r_n\)。若 \(n&lt;2\)，vol20=0；否则 \(\mu=\frac{1}{n}\sum r_k\)，vol20=\(\sqrt{\frac{1}{n}\sum (r_k-\mu)^2}\)（**总体标准差**，除以 \(n\)）。
- **滚动分位数 p**：在至多 756 个历史 raw 组成的窗口内，设小于当前值的个数为 less、等于当前值的个数为 eq、窗口长为 L，则 \(p=(\mathrm{less}+0.5\times\mathrm{eq})/L\)。

## 3.1 趋势因子（权重 40%）

对每个指数 \(i\) 在交易日 \(t\)：

1. `trend_a = close_t / MA20_t - 1`
2. `trend_b = MA20_t / MA60_t - 1`

单指数趋势原始分：

`trend_raw_i = 0.6 * trend_a + 0.4 * trend_b`

四指数合成趋势原始分：

`trend_raw = Σ(0.25 * trend_raw_i)`

## 3.2 量能因子（权重 30%）

对每个指数 \(i\)：

1. `liq_a = amount_t / MA20(amount)_t`
2. `liq_b = vol_t / MA20(vol)_t`（若 vol 缺失，仅使用 liq_a）

单指数量能原始分：

`liq_raw_i = 0.7 * liq_a + 0.3 * liq_b`

四指数合成量能原始分：

`liq_raw = Σ(0.25 * liq_raw_i)`

## 3.3 风险因子（权重 30%，反向）

对每个指数 \(i\)：

1. 日收益：`r_t = close_t / close_t-1 - 1`
2. 20日波动率：`vol20 = std(r_t-19 ... r_t)`
3. 当日振幅：`amp = (high_t - low_t) / close_t`

单指数风险原始分（风险越大值越高）：

`risk_raw_i = 0.7 * vol20 + 0.3 * amp`

四指数合成风险原始分：

`risk_raw = Σ(0.25 * risk_raw_i)`

风险因子为反向得分，后续映射时将“高风险 -> 低分”。

---

## 4. 原始值映射到 0~100

使用滚动分位映射，保证不同阶段可比：

1. 取最近 756 个交易日（约 3 年）同因子历史值组成窗口；
2. 计算当前值在窗口中的分位数 `p`（0~1），算法同 3.0 节「滚动分位数」；
3. 正向因子得分：`score = p * 100`；
4. 风险因子得分：`score = (1 - p) * 100`；
5. 若窗口不足 756 日，使用可用历史窗口，且记录 warmup 标记。

输出：

- `trend_score`（0~100）
- `liquidity_score`（0~100）
- `risk_score`（0~100）

---

## 5. 总温度与分级

## 5.1 总温度

`temperature_score = 0.4 * trend_score + 0.3 * liquidity_score + 0.3 * risk_score`

保留两位小数，区间 [0, 100]。

## 5.2 温度档位（5档固定）

- 极冷：`[0, 20]`
- 偏冷：`(20, 40]`
- 中性：`(40, 60]`
- 偏热：`(60, 80]`
- 过热：`(80, 100]`

## 5.3 方向标记

- `delta_score = today_score - prev_trade_day_score`
- `delta_score >= 2`：升温
- `delta_score <= -2`：降温
- 其他：持平

---

## 6. 策略提示映射（首页悬浮文案）

- 极冷：建议降低总仓位，优先防守
- 偏冷：建议控制仓位，等待确认信号
- 中性：可适度进攻，保持仓位平衡
- 偏热：可积极但注意节奏与止盈
- 过热：警惕回撤风险，避免盲目追高

---

## 7. 增量计算与回补规则

## 7.1 每日自动任务

1. 拉取四指数最新日线并 upsert 到 `market_index_daily_quote`
2. 从“最后已完成交易日”向前回看 80 日，到最新交易日范围重算
3. 将结果写入：
   - `market_temperature_daily`
   - `market_temperature_factor_daily`
4. 唯一键冲突时按 `(trade_date, formula_version)` 覆盖更新

## 7.2 手动重算

支持指定日期区间重算，逻辑同自动任务；可选是否强制刷新行情源。

## 7.3 异常处理

- 当日行情缺失：记录任务失败并重试
- 重试后仍失败：该日结果标记 `failed`
- 首页兜底：返回最近有效交易日并标记 `stale`

---

## 8. 版本管理规则

- 本文档对应版本：`v1.0.0`
- 任一项变化（指数池、因子、权重、分级边界、映射规则）必须升级 `formula_version`
- 新版本不得覆盖旧版本历史结果，必须并存以支持回测对比
