# Feature Specification: 综合选股（股票筛选器）

**Feature Branch**: `001-stock-screener`  
**Created**: 2025-03-15  
**Status**: Draft  
**Input**: User description: "综合选股，帮我生出文件"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看全市场股票列表并分页 (Priority: P1)

用户打开综合选股功能后，能看到 A 股市场全部股票的列表，以表格形式展示，支持分页浏览。表格展示股票的核心识别与行情字段，便于快速浏览与后续筛选。

**Why this priority**: 列表与分页是选股的基础能力，无此则无法进行任何筛选与对比。

**Independent Test**: 进入选股页、确认列表加载出股票数据、切换页码或每页条数，能独立验证“可浏览全市场股票”这一价值。

**Acceptance Scenarios**:

1. **Given** 用户已登录，**When** 用户进入综合选股页面，**Then** 系统展示 A 股股票列表，且支持分页（如每页 20/50/100 条可调）。
2. **Given** 列表已展示，**When** 用户切换页码或每页条数，**Then** 列表更新为对应页的数据，无报错。
3. **Given** 列表已展示，**When** 用户查看表格，**Then** 表格包含可识别的股票标识（如代码、名称）及与产品定义一致的行情/财务等字段。

---

### User Story 2 - 按股票范围筛选 (Priority: P2)

用户可按“股票范围”维度缩小候选池，包括：市场、行业、地区、概念、风格、指数成份、上市时间。选择条件后，列表只显示满足条件的股票。

**Why this priority**: 范围筛选是选股最常用的第一步，直接影响后续分析的工作量。

**Independent Test**: 选择任意一个或多个范围条件（如某行业、某市场），确认列表仅显示符合条件的结果，且结果数量与预期一致。

**Acceptance Scenarios**:

1. **Given** 用户在选股页，**When** 用户选择市场/行业/地区/概念/风格/指数成份/上市时间中的至少一个条件，**Then** 列表仅展示满足所选条件的股票。
2. **Given** 用户已设置多个范围条件，**When** 用户清除某一条件，**Then** 列表随之更新，包含更多或更少股票。
3. **Given** 用户未选任何范围条件，**When** 列表加载，**Then** 行为与“全市场”一致，展示全部 A 股（或当前数据源覆盖的股票）。

---

### User Story 3 - 按基本面 / 技术面 / 消息面 / 人气 / 行情筛选 (Priority: P3)

用户除范围外，还能按以下维度进一步筛选：

- **基本面**：估值指标、每股指标、盈利能力、成长能力、资本结构与偿债能力、股本股东。
- **技术面**：MACD 金叉、KDJ 金叉、放量突破、低位资金净流入、高位资金净流出、向上突破均线、均线多头/空头排列、连涨放量、下跌无量、一根/两根大阳线、旭日东升、强势多方、炮拨云见日、七仙女下凡(七连阴)、八仙过海(八连阳)、九阳神功(九连阳)、四串阳、天量法则、放量上攻、穿头破脚、倒转锤头、射击之星、黄昏之星、曙光初现、身怀六甲、乌云盖顶、早晨之星、窄幅整理等形态与指标条件。
- **消息面**：公告大事、机构关注情况、机构持股家数、机构持股比例。
- **人气指标**：股吧人气排名、人气排名变化、人气排名连涨/连跌、人气排名创新高/新低、新晋粉丝占比、铁杆粉丝占比、7 日关注排名、今日浏览排名。
- **行情数据**：股价表现、成交情况、资金流向、行情统计、沪深股通。

选择后，列表只显示同时满足“股票范围”与上述所选条件的股票。

**Why this priority**: 多维度筛选是“综合选股”的核心价值，在列表与范围可用之后实现。

**Independent Test**: 在已选范围（或全市场）下，勾选或设置若干基本面/技术面/消息面/人气/行情条件，确认结果集正确缩小且可复现。

**Acceptance Scenarios**:

1. **Given** 用户已在选股页并可选范围，**When** 用户设置任意基本面、技术面、消息面、人气或行情条件，**Then** 列表仅展示同时满足范围与这些条件的股票。
2. **Given** 用户已设置多类条件，**When** 用户修改或取消某一条件，**Then** 列表实时或经刷新后更新，结果与当前条件集一致。
3. **Given** 用户设置的条件组合过严，**When** 无股票满足，**Then** 列表为空，并有明确提示（如“暂无符合条件股票”），不报错。

---

### Edge Cases

- 当数据源暂时无某只股票或某维度数据缺失时，该股票在依赖该维度的筛选中应被排除或明确标注，不导致整页报错。
- 当用户选择的条件组合导致结果集为空时，系统展示空状态提示，并允许用户放宽条件。
- 当结果集很大（如数千只）时，分页与筛选响应在可接受时间内完成（具体性能目标见 Success Criteria）。
- 当用户未选任何筛选条件时，列表等价于“全市场”，不因缺省参数而报错或空白。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须展示 A 股市场股票列表，并支持分页（每页条数可配置，如 20/50/100）。
- **FR-002**: 系统必须支持按“股票范围”筛选：市场、行业、地区、概念、风格、指数成份、上市时间。
- **FR-003**: 系统必须支持按“基本面”相关条件筛选（估值指标、每股指标、盈利能力、成长能力、资本结构与偿债能力、股本股东）。
- **FR-004**: 系统必须支持按“技术面”条件筛选（如 MACD/KDJ 金叉、放量突破、资金净流入/流出、均线相关、指定 K 线形态等产品能力中列出的项）。
- **FR-005**: 系统必须支持按“消息面”筛选：公告大事、机构关注情况、机构持股家数、机构持股比例。
- **FR-006**: 系统必须支持按“人气指标”筛选（股吧人气排名、排名变化/连涨/连跌、创新高/新低、粉丝占比、关注/浏览排名等）。
- **FR-007**: 系统必须支持按“行情数据”筛选：股价表现、成交情况、资金流向、行情统计、沪深股通。
- **FR-008**: 多维度条件同时生效时，结果必须为同时满足“股票范围”与用户所选基本面/技术面/消息面/人气/行情条件的交集。
- **FR-009**: 列表表格必须包含产品定义中约定的字段（股票标识及所选维度对应的展示字段），以便用户识别股票并判断是否符合预期。
- **FR-010**: 当无股票满足当前条件时，系统必须展示明确空状态提示，不暴露技术错误信息。

### Key Entities

- **股票（列表项）**：代表一只 A 股在筛选结果中的一行，包含股票标识（如代码、名称）以及用于展示与筛选的维度属性（范围、基本面、技术面、消息面、人气、行情等）。
- **筛选条件**：用户选择的股票范围、基本面、技术面、消息面、人气、行情等条件的集合；条件可多选或区间设置，组合结果为交集。
- **股票范围维度**：市场、行业、地区、概念、风格、指数成份、上市时间等分类属性。
- **指标/形态维度**：基本面指标、技术指标与形态、消息与机构数据、人气指标、行情与资金数据等，用于筛选与表格展示。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户能在进入选股页后，在合理时间内（如 5 秒内）看到首屏股票列表并正常分页。
- **SC-002**: 用户通过范围 + 至少一类其他维度（基本面/技术面/消息面/人气/行情）的组合筛选，能得到与条件一致的结果集，且结果可复现。
- **SC-003**: 当条件过严导致无结果时，用户能看到明确空状态说明，且能通过放宽条件再次得到列表。
- **SC-004**: 列表与筛选行为满足“综合选股”产品定义中的六大类维度（股票范围、基本面、技术面、消息面、人气、行情），无遗漏维度类别。

## Assumptions

- 股票数据来源由系统或后续实现统一提供；本规格不限定具体数据接口或存储方式。
- 默认针对 A 股市场；若数据源包含其他市场，可通过“市场”范围筛选限定。
- 技术面、人气等条件的具体计算口径（如“金叉”的周期、人气排名的统计范围）以产品与数据源约定为准，本规格仅要求“支持按这些维度筛选”。
- 用户为已登录的本地部署用户，权限与登录能力由现有用户模块保证。
