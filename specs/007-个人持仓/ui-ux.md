# 前端交互与视觉设计：个人持仓

**功能**: 007-个人持仓 | **日期**: 2026-03-22 | **关联**: [plan.md](./plan.md)、[spec.md](./spec.md)

**定位**：个人持仓是用户**高频查看、反复复盘**的页面，设计目标为：**信息一眼能扫清、操作路径短、长时间阅读不累、视觉沉稳可信**（偏「工作台」而非营销页）。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **层级清晰** | 先总览（持仓盈亏、胜率卡片）→ 再列表 → 再单笔详情与操作时间线；避免首屏塞满表单。 |
| **复盘友好** | 已完结交易以「时间 + 标的」可检索；详情内**操作时间线**自上而下展示每一次建仓/加减仓/清仓；复盘文字与图片**同屏可达**，减少跳转。 |
| **情绪克制** | 盈亏用颜色区分（红涨绿跌或沿用 A 股习惯），避免大面积刺眼色块；数字tabular对齐。 |
| **与全站一致** | 沿用 `Layout` 侧栏、`el-card` 白底、`#f5f7fa` 页面底、主色与 Logo 区 **#1e3a5f** 同系；与 [StockScreeningView](../) 的表格 + 卡片头风格对齐。 |
| **能力说明不打扰** | 标题旁 **「?」或「查看说明」** 用 `el-tooltip` / `el-popover`（与综合选股「查看当前产品能力」同模式），**默认折叠**。 |

---

## 2. 信息架构

- **一级菜单「个人服务」** → **二级「个人持仓」** 进入**单页多区块**（推荐 **Tab** 或 **锚点**，避免多路由碎片）：
  1. **概览**：今日/全局摘要卡片（可选）+ 胜率卡片 + 快捷入口。
  2. **当前持仓**：表格 + 行内「加仓 / 减仓 / 清仓」。
  3. **已完结**：可筛选列表（代码、日期区间）→ 点进**详情抽屉 / 侧栏**做复盘。
  4. **（可选同一页底部）** 操作记录总览：按时间倒序的「最近操作」便于复盘。

**推荐结构（单路由 `/personal-holdings`）**：

```text
[ 页面标题 + 能力说明图标 ]
[ Tab: 概览 | 当前持仓 | 已完结与复盘 | 统计 ]

概览 Tab：胜率环形/条形 + 简要数字 + 引导去持仓/已完结
当前持仓 Tab：表格
已完结 Tab：列表 → 点击 → Drawer 详情（时间线 + 复盘 + 图）
统计 Tab：股票胜率 / 操作胜率拆开展示 + 分母说明文案
```

若首版希望简单，可 **合并「概览」与「统计」** 为一个 Tab「总览」，减少一次点击。

---

## 3. 页面布局要点

### 3.1 页头

- 标题：**个人持仓**；副标题或灰色小字：**管理持仓、记录操作与复盘（不构成投资建议）**。
- 右侧或标题行末：`el-tooltip` / `el-popover`，说明：
  - 一笔交易 = 建仓至清仓；
  - 加仓/减仓为操作记录；
  - 股票胜率看整笔盈亏，操作胜率看每次操作自评。

### 3.2 当前持仓区

- **表格**：`el-table` + `stripe`，列含：代码、名称、持仓数量、成本价、参考价、参考市值、参考盈亏、盈亏幅度、操作。
- **盈亏列**：`>=0` 与 `<0` 用 **class** 区分颜色（建议：**盈** 用主题绿系 `#16a34a` 或沿用 Element `success`；**亏** 用克制红 `#dc2626`，避免荧光）。
- **行操作**：主按钮「加仓」「减仓」；危险按钮「清仓」需 **二次确认**（`ElMessageBox`）。
- **空状态**：`el-empty` + 插图轻量或纯文案：**「暂无持仓」**，主按钮「建仓」。

### 3.3 已完结与复盘区（核心体验）

- **列表**：卡片列表或表格；每行展示：代码、名称、清仓日、**整笔盈亏**（同色规则）、是否有复盘/图片角标。
- **进入详情**：**右侧 Drawer**（宽度 `min(560px, 90vw)`）优于全屏跳转，便于对照列表。
- **Drawer 内布局（上 → 下）**：
  1. **头部**：标的、清仓日、整笔盈亏大号数字。
  2. **操作时间线**：`el-timeline` 展示每条 `portfolio_operation`：日期、类型（建仓/加仓/减仓/清仓）、数量、价格、**操作自评**（好/未评/坏，可用 `el-tag` + 行内快捷修改）。
  3. **复盘区**：标题「复盘」；多行文本 `el-input type="textarea"` 或轻量 Markdown（首版纯文本即可）；**图片墙**：`el-upload` 列表 + 缩略图网格，支持预览（`el-image` + `preview-src-list`）。
  4. **底部固定**：保存复盘、关闭 Drawer。

### 3.4 统计区

- **两个并排卡片**（`el-row` + `el-col`）：
  - **股票胜率**：环形图或「胜/负/平」数字 + 百分比条形；小字说明分母为「已完结交易笔数」。
  - **操作胜率**：同上；小字说明「仅统计已评价操作，未评价不计入分母」。
- 避免一个图表混写两种胜率，**物理分开**以防误解（与规格一致）。

## 4. 视觉与样式

| 项 | 建议 |
|----|------|
| **页面背景** | `#f5f7fa`（与 `Layout` 一致）。 |
| **卡片** | `el-card shadow="never"`，白底，圆角与全站统一；卡片头加 **底部分割线** 或浅灰背景区分。 |
| **主色** | 链接、主按钮：`#409eff`（Element 默认）或略调深以贴近 `#1e3a5f` 的蓝色系。 |
| **字体** | 沿用系统栈；数字用 `font-variant-numeric: tabular-nums` 便于对齐。 |
| **间距** | 卡片内 `16–24px`；区块之间 `24px`；避免拥挤。 |
| **动效** | 加载用 `v-loading`；Drawer 使用默认过渡即可；**禁止**花哨全屏动画。 |

### 4.1 可选微质感（在不过度前提下）

- 胜率卡片背景：**极浅蓝灰** `linear-gradient` 或 `#f8fafc`。
- 时间线当前无特殊高亮需求；清仓节点可用 **outline** 弱强调。

## 5. 关键交互

| 场景 | 行为 |
|------|------|
| 建仓 | 弹窗表单：代码（可 Autocomplete 关联 `stock_basic`）、日期、数量、价格、手续费；提交成功 → Toast + 刷新列表。 |
| 加仓/减仓 | 弹窗仅填增量字段；减仓校验不超过持仓。 |
| 清仓 | 强确认文案：**「清仓后将结束本笔交易并生成已完结记录」**；确认后跳转或提示去「已完结」补复盘。 |
| 操作自评 | 时间线每行下拉或 Tag 切换「好 / 坏 / 未评」；**自动保存**或显式保存二选一（推荐显式「保存自评」防误触，或 debounce 自动保存）。 |
| 图片上传 | 限制数量与大小前端校验；失败明确提示；上传中按钮 loading。 |
| 错误与空数据 | 接口失败 `ElMessage.error`；无日线价时参考市值显示 **「—」** 与 `tooltip` 说明「暂无同步行情」。 |

## 6. 无障碍与习惯

- 主要按钮与危险操作可用 **键盘 Tab** 聚焦（Element 默认较好）。
- 图表若使用 ECharts，保留 **标题与数据摘要** 文本，避免仅依赖图形。

## 7. 实现时文件建议

- 视图：`frontend/src/views/PersonalHoldingsView.vue`（可拆子组件 `HoldingsOverview.vue`、`OpenPositionsTable.vue`、`ClosedTradesList.vue`、`TradeReviewDrawer.vue`、`WinRateCards.vue`）。

## 8. 与规格、契约的对应关系

- **操作记录**：时间线 ↔ `portfolio_operation`。
- **整笔复盘**：Drawer 内复盘区 ↔ `portfolio_trade.review_text` + `portfolio_trade_image`。
- **胜率**：统计 Tab ↔ `GET /api/portfolio/stats`（见 [contracts/personal-portfolio-api.md](./contracts/personal-portfolio-api.md)）。

本文档为**设计与验收参考**；实现时可按迭代裁剪，但**复盘 Drawer + 时间线 + 双胜率分区**建议作为 MVP 体验基线。
