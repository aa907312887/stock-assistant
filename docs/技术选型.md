# 技术选型说明

## 项目目标

与 [产品能力](产品能力.md) 对齐，支撑以下核心能力：

- **用户模块**：本地部署，简单登录（用户名 + 统一密码，或仅用户名）；无注册入口，用户由 DB 预置；用户 ID 唯一，用于后续按用户维度分析持仓。
- **综合选股**：A 股全市场股票列表，分页表格展示；支持多维度筛选（股票范围、基本面、技术面、消息面、人气指标、行情数据）。
- **持仓助手**：用户录入持仓，通过数据接口查询持仓标的，为后续分析与测算提供数据基础。
- **数据与调度**：对接外部股票数据接口，定时拉取与计算（不使用缓存）。
- **可扩展**：预留 AI 能力（选股建议、复盘、问答等）及回测/策略研究扩展。
- **体验**：前端界面美观、易用，接口清晰、可维护。

## 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         Frontend (Vue 3)                         │
│  页面 / 图表 / 状态管理 / API 调用                                 │
└─────────────────────────────┬───────────────────────────────────┘
                              │ HTTP / WebSocket（可选）
┌─────────────────────────────▼───────────────────────────────────┐
│                      Backend (FastAPI)                           │
│  API 路由 / 业务逻辑 / 数据服务 / 定时任务 / （未来）AI 服务         │
└─────┬───────────────┬───────────────┬────────────────────────────┘
      │               │               │
      ▼               ▼               ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ MySQL        │   │ 股票数据接口  │   │ 任务调度器   │
│(SQLAlchemy)  │   │（可插拔实现） │   │(APScheduler) │
│ 业务数据持久化│   │ 后续由你提供  │   │ 定时拉数/算指标│
└──────────────┘   └──────────────┘   └──────────────┘
```

设计原则：

- **数据层与数据源解耦**：股票数据通过抽象接口接入，你后续提供的接口实现即可替换或并存。
- **定时任务独立**：用调度器定时拉数据、算指标，不阻塞 API。
- **AI 预留**：业务层按「分析服务」抽象，后续在相同接口下挂载 AI 模型或外部 API。

## 后端：Python 3.12 + FastAPI

### 核心栈

| 类别     | 选型              | 说明 |
|----------|-------------------|------|
| 语言     | Python 3.12       | 与当前环境一致，量化与数据生态成熟。 |
| Web 框架 | FastAPI           | 异步、自动 OpenAPI 文档、类型提示友好，便于与前端协作。 |
| 数据库   | SQLAlchemy + MySQL | 用户（本地预置、无注册）、用户持仓、可选同步的股票/行业/概念等基础数据；不使用缓存。 |

### 数据与量化

- **pandas / numpy**：行情与指标计算、综合选股筛选逻辑、持仓数据聚合。
- **TA-Lib**（或 `ta`、`pandas-ta`）：技术指标与形态，支撑综合选股中的技术面筛选（如均线、MACD/KDJ 金叉、多种 K 线形态等）。
- **股票数据接口**：抽象为「数据源适配层」，需覆盖产品能力中的全部数据需求：
  - **股票列表与维度**：全市场股票、分页；股票范围（市场、行业、地区、概念、风格、指数成份、上市时间）。
  - **基本面**：估值、每股指标、盈利能力、成长能力、资本结构与偿债、股本股东等。
  - **技术面**：K 线数据 + 本地计算形态/指标（MACD/KDJ 金叉、均线多空排列、量价形态等）。
  - **消息面**：公告大事、机构关注、机构持股家数/比例。
  - **人气指标**：股吧人气、排名、粉丝占比、关注/浏览排名等。
  - **行情**：股价表现、成交、资金流向、行情统计、沪深股通。
  - **持仓**：根据用户持仓列表查询标的行情与指标。
  - 你后续提供的数据接口（HTTP/SDK/文件等）实现该抽象，便于切换或扩展多数据源。

### 认证

- **简单登录**：用户名 + 统一密码（或仅用户名），Session/JWT 二选一；无注册、无找回密码，用户仅在 MySQL 中预置，用户 ID 唯一。
- 鉴权中间件保护选股、持仓等接口，按用户 ID 隔离持仓数据。

### 定时任务

- **APScheduler**  
  - 进程内调度，部署简单，适合：定时拉取全市场/行业/概念等基础数据、行情与指标，供综合选股与持仓查询使用；按需生成报表。  
  - 与 FastAPI 生命周期集成（`lifespan` 内启动/关闭），无需额外 Worker。
- 不使用 Redis 或其它缓存/队列中间件；若日后任务量极大再考虑引入 Celery 等分布式方案。

### AI 扩展

- 在 Python 内预留「分析/建议」服务层，接口与现有业务一致。
- 后续可按需接入：**LangChain**、**OpenAI API**、本地模型或其它 Agent 框架，不绑定具体实现。

## 前端：Vue 3 + TypeScript

### 核心栈

| 类别     | 选型              | 说明 |
|----------|-------------------|------|
| 框架     | Vue 3             | Composition API、生态成熟，与个人使用习惯一致。 |
| 语言     | TypeScript        | 类型安全，与后端 API 协作更清晰。 |
| 构建     | Vite              | 开发体验与构建速度好。 |
| UI 组件  | Element Plus / Naive UI | 按页面需求选择，保证界面美观、易用。 |
| 图表     | ECharts           | K 线、收益曲线、账户分析、指标展示。 |
| 表格/分页 | 与 UI 组件库配套   | 综合选股大表、分页、排序、列配置。 |
| 状态/请求| Pinia + 封装 axios/fetch | 用户登录态、全局状态与 API 调用统一管理。 |

### 与后端协作

- 通过 OpenAPI 生成或手写 TypeScript 类型，与 FastAPI 契约一致。
- 登录态（Session/JWT）在请求头或 Cookie 中携带，前端需维护登录页、登出与鉴权失败处理。
- 综合选股：表格分页、多条件筛选与后端选股 API 对接；持仓助手：持仓列表与录入表单与持仓 API 对接。
- 后续若增加 AI 对话/建议，可沿用同一套 API 与状态管理，仅增加新页面与组件。

## 模块划分（后端）

与产品能力对应，建议在后端中区分：

| 模块           | 职责 |
|----------------|------|
| **用户/认证**  | 简单登录（用户名+统一密码或仅用户名）、Session/JWT、鉴权中间件；用户表在 MySQL 预置，无注册。 |
| **API 路由**   | 对外 HTTP 接口，参数校验、鉴权；按用户 ID 隔离持仓等数据。 |
| **综合选股**   | 全市场股票列表分页；多维度筛选（股票范围、基本面、技术面、消息面、人气、行情）组装与查询。 |
| **持仓助手**   | 用户持仓的 CRUD、与数据接口对接查询持仓标的行情与指标，为后续分析预留接口。 |
| **数据服务**   | 调用「股票数据接口」、技术指标与形态计算；不依赖缓存。 |
| **数据源适配** | 实现你提供的股票数据接口，覆盖产品能力所需维度（列表、范围、基本面、技术面、消息、人气、行情、持仓查询）。 |
| **定时任务**   | 定时拉基础数据、行情与指标，供选股与持仓使用。 |
| **AI 服务**    | 预留；后续实现选股建议、复盘、问答等。 |
