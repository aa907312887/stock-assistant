# 数据库设计

本文档基于 [产品能力](产品能力.md) 与 [技术选型](技术选型.md) 进行数据库设计。数据库采用 **MySQL**，ORM 使用 **SQLAlchemy**。

---

## 设计说明

- **核心表**：用户、用户持仓、**投资逻辑条目**（`investment_logic_entry`），与产品功能强相关，必须落库。
- **股票基础表 `stock_basic`**：仅存放**相对静态**的股票身份与范围信息（代码、名称、市场、行业、地区、上市日期等），**不存放**每日变动的行情（价、量）和业绩/估值（市盈率、ROE、毛利率等）。
- **行情、业绩/估值**：拆分为**日行情表**（按日一条）+ **日估值表**（按交易日）+ **财报表**（按报告期），避免口径混用。
- **数据新鲜度可审计**：核心同步表统一增加 `data_source`、`sync_batch_id`、`synced_at`、`extra_json`，便于判断“今天数据是否完整”。
- **扩展策略**：优先“结构化字段 + `extra_json`”，避免盲目预留大量空列，新增接口字段可先落 `extra_json` 再结构化迁移。
- **可选同步表**：股票基础、行业/概念/风格/指数成份、行情、业绩估值等，若综合选股完全依赖外部接口可不建或仅部分使用。
- **命名**：表名小写 + 下划线；主键统一 `id`（BIGINT 自增）；时间字段 `created_at` / `updated_at`（DATETIME）。
- **数据库名**：`stock_assistant`（与项目一致，UTF-8 存储）。

## 1. 用户模块

### 1.1 用户表 `user`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键，唯一标识用户 |
| username | VARCHAR(64) | UNIQUE, NOT NULL | 登录用户名，唯一 |
| password_hash | VARCHAR(255) | NULL | 密码哈希；若采用「仅用户名」登录可留空或统一固定值 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 无注册入口，用户仅在 DB 中预置；后续按 `user_id` 分析持仓。
- 索引：`username` 已唯一，可作登录查询。

### 1.2 投资逻辑条目表 `investment_logic_entry`

用户投资逻辑**历史快照**：按 **技术面 / 基本面 / 消息面** 三面存储正文，并存储三面**权重**（整数百分比，三者之和为 100）。一名用户可有多条，用于首页展示「当前」与历史回顾（业务规则见 `specs/004-首页投资逻辑/data-model.md`）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | NOT NULL, FK → user.id | 用户 ID |
| technical_content | TEXT | NULL | 技术面正文 |
| fundamental_content | TEXT | NULL | 基本面正文 |
| message_content | TEXT | NULL | 消息面正文 |
| weight_technical | TINYINT UNSIGNED | NOT NULL | 技术面权重（0–100） |
| weight_fundamental | TINYINT UNSIGNED | NOT NULL | 基本面权重 |
| weight_message | TINYINT UNSIGNED | NOT NULL | 消息面权重 |
| created_at | DATETIME | NOT NULL | 本条新增时间 |
| updated_at | DATETIME | NOT NULL | 本条最后修改时间 |
| extra_json | JSON | NULL | 扩展字段；约定 **`insights`**（`string[]`）存「重要感悟」第1点、第2点…，其它键可后续扩展 |

- 索引：`user_id`；删除用户时可 `ON DELETE CASCADE` 或按产品选择保留策略（默认与 FK 一致）。
- **不在** `user` 表存放整段投资逻辑；与首页投资逻辑规格一致。

## 2. 持仓助手

### 2.1 用户持仓表 `user_position`

持仓汇总表：同一用户同一只股票一条记录，展示当前持仓、成本、**当前收益额/收益率**（由最新行情更新或接口查询时计算写入）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | NOT NULL, FK → user.id | 用户 ID |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码（如 000001.SZ、600000.SH） |
| stock_name | VARCHAR(100) | NULL | 股票名称（冗余，便于列表展示；也可由接口补全） |
| quantity | DECIMAL(20,4) | NOT NULL | 持仓数量（股） |
| cost_price | DECIMAL(12,4) | NULL | 成本价（持仓成本均价） |
| **profit_amount** | **DECIMAL(16,4)** | **NULL** | **当前收益额（元），(现价－成本价)×数量，随行情更新** |
| **yield_rate** | **DECIMAL(10,4)** | **NULL** | **当前收益率（%），收益额/成本金额×100，随行情更新** |
| memo | VARCHAR(500) | NULL | 备注 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 约束：`UNIQUE(user_id, stock_code)`，同一用户同一只股票一条汇总记录。
- `profit_amount`、`yield_rate` 可由定时任务或查询时根据最新行情（如 `stock_daily_quote` 最新日收盘价）计算后更新。
- 索引：`user_id`（查某用户持仓）、`(user_id, stock_code)`（唯一约束已覆盖）。

### 2.2 用户交易明细表 `user_trade`

记录每笔**买入、卖出**，用于展示「购买时间列表」「卖出时间列表」，并可与持仓汇总表联动（新增/减少持仓、成本重算）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | NOT NULL, FK → user.id | 用户 ID |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| trade_type | VARCHAR(10) | NOT NULL | 交易类型：`buy` 买入 / `sell` 卖出 |
| quantity | DECIMAL(20,4) | NOT NULL | 成交数量（股） |
| price | DECIMAL(12,4) | NOT NULL | 成交价格 |
| trade_time | DATETIME | NOT NULL | 交易时间（即该笔的购买时间或卖出时间） |
| amount | DECIMAL(16,2) | NULL | 成交金额（可选，price×quantity） |
| memo | VARCHAR(500) | NULL | 备注 |
| created_at | DATETIME | NOT NULL | 创建时间 |

- 购买时间列表：`WHERE user_id=? AND stock_code=? AND trade_type='buy' ORDER BY trade_time`。
- 卖出时间列表：`WHERE user_id=? AND stock_code=? AND trade_type='sell' ORDER BY trade_time`。
- 业务上新增一条买入/卖出时，需同步更新或重算 `user_position` 的 `quantity`、`cost_price`（及可选 `profit_amount`、`yield_rate`）。
- 索引：`(user_id, stock_code, trade_type)`、`trade_time`。

## 3. 综合选股（可选同步）

以下表用于在本地存储股票及维度数据，支撑综合选股的**股票范围**筛选（市场、行业、地区、概念、风格、指数成份、上市时间）及列表展示。若选股数据全部由外部接口提供，可仅保留「用户 + 持仓」表，本小节表为可选。

### 3.1 股票基础表 `stock_basic`

**仅存放相对静态的股票身份与范围信息，不存放股价、成交量、市盈率、ROE、毛利率等**（这些放在行情表、业绩估值表中）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| code | VARCHAR(20) | UNIQUE, NOT NULL | 股票代码 |
| name | VARCHAR(100) | NULL | 股票名称 |
| market | VARCHAR(20) | NULL | 市场（如 沪市/深市/北交所） |
| industry_code | VARCHAR(20) | NULL | 行业编码，关联行业维度 |
| industry_name | VARCHAR(100) | NULL | 行业名称（冗余） |
| region | VARCHAR(50) | NULL | 地区 |
| list_date | DATE | NULL | 上市日期 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 索引：`code`（唯一）、`market`、`industry_code`、`list_date`（便于范围筛选）。

### 3.2 行业维度表 `dict_industry`（可选）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| code | VARCHAR(20) | UNIQUE, NOT NULL | 行业编码 |
| name | VARCHAR(100) | NOT NULL | 行业名称 |
| sort_order | INT | NULL | 排序 |

### 3.3 股票-概念关联表 `stock_concept`（多对多，可选）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| concept_code | VARCHAR(50) | NULL | 概念编码 |
| concept_name | VARCHAR(100) | NULL | 概念名称 |

- 索引：`stock_code`、`concept_name`（按概念筛选）。

### 3.4 股票-风格关联表 `stock_style`（可选）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| style_code | VARCHAR(50) | NULL | 风格编码 |
| style_name | VARCHAR(100) | NULL | 风格名称 |

### 3.5 指数成份表 `stock_index_member`（可选）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| index_code | VARCHAR(20) | NOT NULL | 指数代码 |
| index_name | VARCHAR(100) | NULL | 指数名称 |
| stock_code | VARCHAR(20) | NOT NULL | 成份股代码 |
| weight | DECIMAL(10,4) | NULL | 权重（若有） |

- 索引：`index_code`、`stock_code`。

### 3.6 股票日行情表 `stock_daily_quote`

存放**每日行情**：OHLCV、涨跌幅、内盘外盘、委比、换手率等。按「股票 + 交易日期」一条记录，用于 K 线、选股中的行情筛选、持仓成本对比等。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| trade_date | DATE | NOT NULL | 交易日期 |
| open | DECIMAL(12,4) | NULL | 开盘价 |
| close | DECIMAL(12,4) | NULL | 收盘价 |
| high | DECIMAL(12,4) | NULL | 最高价 |
| low | DECIMAL(12,4) | NULL | 最低价 |
| prev_close | DECIMAL(12,4) | NULL | 前收盘价 |
| change_amount | DECIMAL(12,4) | NULL | 涨跌额 |
| pct_change | DECIMAL(10,4) | NULL | 涨跌幅（%） |
| volume | DECIMAL(20,2) | NULL | 成交量 |
| amount | DECIMAL(20,2) | NULL | 成交额 |
| amplitude | DECIMAL(10,4) | NULL | 振幅（%） |
| turnover_rate | DECIMAL(10,4) | NULL | 换手率（%） |
| volume_ratio | DECIMAL(10,4) | NULL | 量比 |
| internal_volume | DECIMAL(20,2) | NULL | 内盘（主动卖出成交量） |
| external_volume | DECIMAL(20,2) | NULL | 外盘（主动买入成交量） |
| bid_volume | DECIMAL(20,2) | NULL | 委买量 |
| ask_volume | DECIMAL(20,2) | NULL | 委卖量 |
| bid_ask_ratio | DECIMAL(10,4) | NULL | 委比（%） |
| total_market_cap | DECIMAL(20,2) | NULL | 总市值 |
| float_market_cap | DECIMAL(20,2) | NULL | 流通市值 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 约束：`UNIQUE(stock_code, trade_date)`。
- 索引：`stock_code`、`trade_date`、`(stock_code, trade_date)`。

### 3.7 股票每日估值表 `stock_valuation_daily`

存放**按交易日的估值与部分基本面快照**，用于综合选股列表和当日筛选。该表与 `stock_daily_quote` 同频（日级）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| trade_date | DATE | NOT NULL | 交易日期 |
| pe_ttm | DECIMAL(12,4) | NULL | 市盈率 TTM |
| pe_static | DECIMAL(12,4) | NULL | 市盈率静态 |
| pe_dynamic | DECIMAL(12,4) | NULL | 市盈率动态 |
| pe | DECIMAL(12,4) | NULL | 市盈率通用口径 |
| pb | DECIMAL(12,4) | NULL | 市净率 |
| ps | DECIMAL(12,4) | NULL | 市销率 |
| roe | DECIMAL(10,4) | NULL | ROE（%） |
| gross_margin | DECIMAL(10,4) | NULL | 毛利率（%） |
| net_margin | DECIMAL(10,4) | NULL | 净利率（%） |
| data_source | VARCHAR(32) | NOT NULL | 数据源（如 tushare） |
| sync_batch_id | VARCHAR(64) | NULL | 同步批次号（用于追踪） |
| synced_at | DATETIME | NOT NULL | 同步时间 |
| extra_json | JSON | NULL | 扩展字段（临时承载新增指标） |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 约束：`UNIQUE(stock_code, trade_date)`，严格保证同股同日一条。
- 索引：`(trade_date, stock_code)`、`(trade_date, pe)`、`(trade_date, pb)`、`(trade_date, roe)`。

### 3.8 股票财报表 `stock_financial_report`

存放**按报告期（季报/年报）**的财务数据历史，用于“营收指标变化”等跨期分析。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |
| report_date | DATE | NOT NULL | 报告期（如 2025-12-31） |
| report_type | VARCHAR(16) | NULL | 报告类型（Q1/H1/Q3/FY） |
| revenue | DECIMAL(20,2) | NULL | 营收 |
| net_profit | DECIMAL(20,2) | NULL | 净利润 |
| eps | DECIMAL(12,4) | NULL | 每股收益 |
| bps | DECIMAL(12,4) | NULL | 每股净资产 |
| roe | DECIMAL(10,4) | NULL | ROE（%） |
| gross_margin | DECIMAL(10,4) | NULL | 毛利率（%） |
| net_margin | DECIMAL(10,4) | NULL | 净利率（%） |
| data_source | VARCHAR(32) | NOT NULL | 数据源 |
| sync_batch_id | VARCHAR(64) | NULL | 同步批次号 |
| synced_at | DATETIME | NOT NULL | 同步时间 |
| extra_json | JSON | NULL | 扩展字段 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 约束：`UNIQUE(stock_code, report_date)`；`report_date` 设为 **NOT NULL**，避免 MySQL `NULL` 唯一键重复问题。
- 索引：`(stock_code, report_date)`、`(report_date)`。

### 3.9 股票每日技术分析表 `stock_daily_tech`

存放**每日技术指标与形态**：K 线形态、MACD、KDJ、均线、乖离率、RSI 等，与 `stock_daily_quote` 一一对应（股票 + 交易日期），由定时任务根据行情计算写入，供综合选股技术面筛选与 K 线分析页展示。

| 字段 | 类型 | 约束 | 说明 |

| stock_code | VARCHAR(20) | NOT NULL | 股票代码 |

| **macd_dif** | **DECIMAL(12,6)** | **NULL** | **MACD DIF** |
| **macd_dea** | **DECIMAL(12,6)** | **NULL** | **MACD DEA** |
| **macd_hist** | **DECIMAL(12,6)** | **NULL** | **MACD 柱（DIF-DEA）** |
| **kdj_k** | **DECIMAL(10,4)** | **NULL** | **KDJ K 值** |
| **kdj_d** | **DECIMAL(10,4)** | **NULL** | **KDJ D 值** |
| **kdj_j** | **DECIMAL(10,4)** | **NULL** | **KDJ J 值** |
| **ma5** | DECIMAL(12,4) | NULL | 5 日均线 |
| **ma10** | DECIMAL(12,4) | NULL | 10 日均线 |
| **ma20** | DECIMAL(12,4) | NULL | 20 日均线 |
| **ma60** | DECIMAL(12,4) | NULL | 60 日均线 |
| **ma_pattern** | **VARCHAR(50)** | **NULL** | **均线形态**（如 多头排列、空头排列等） |
| **bias5** | DECIMAL(10,4) | NULL | 5 日乖离率（%） |
| **bias10** | DECIMAL(10,4) | NULL | 10 日乖离率（%） |
| **bias20** | DECIMAL(10,4) | NULL | 20 日乖离率（%） |
| **bias60** | DECIMAL(10,4) | NULL | 60 日乖离率（%） |
| **rsi** | DECIMAL(8,4) | NULL | RSI（一般 6/12/24 取其一或存多条，此处先存一条） |
| **rsi6** | DECIMAL(8,4) | NULL | RSI(6)（可选） |
| **rsi12** | DECIMAL(8,4) | NULL | RSI(12)（可选） |
| **volume_ma5** | DECIMAL(20,2) | NULL | 5 日成交量均量 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

- 约束：`UNIQUE(stock_code, trade_date)`。
- 索引：`stock_code`、`trade_date`、`(stock_code, trade_date)`；按形态筛选时可对 `kline_pattern`、`ma_pattern` 建索引（视查询再定）。

## 4. 数据来源与扩展说明

| 能力 | 数据来源建议 | 说明 |
|------|--------------|------|
| 用户、持仓 | 仅存 MySQL | 用户预置，持仓为用户录入。 |
| 综合选股 - 股票范围 | 定时同步至 `stock_basic` 等表，或全部走接口 | 若本地筛选则依赖上述可选表。 |
| 行情（价、量、内盘外盘委比等） | 定时同步至 `stock_daily_quote` | OHLCV、涨跌幅、内盘/外盘/委比/换手率/量比/市值等按日写入。 |
| 每日技术分析（K 线形态、MACD、KDJ、均线、乖离率） | 定时计算写入 `stock_daily_tech` | 由 `stock_daily_quote` 计算得出，供技术面筛选与 K 线分析。 |
| 日估值（PE 动/静/TTM、PB、ROE、毛利率等） | 定时同步至 `stock_valuation_daily` | 供“当日综合选股”筛选，口径与交易日一致。 |
| 财报历史（营收/净利润等） | 定时同步至 `stock_financial_report` | 供“营收变化、利润变化”等跨期分析。 |
| 综合选股 - 消息面/人气 | 以外部数据接口为主或本地表 | 可后续按接口增加表或接口直查。 |

- 技术面、消息面、人气等字段随接口或指标定义，可在「数据接口说明」确定后按需增加表或接口直查。

## 6. 建库与建表语句（V2）

为避免文档内维护两套 SQL，V2 采用独立脚本管理：

- **清库重建 + V2 新表**：`backend/scripts/reset_and_init_v2.sql`
- **执行方式**：`mysql -u root -p < backend/scripts/reset_and_init_v2.sql`

该脚本已包含以下内容：

1. 删除旧版所有相关表（`DROP TABLE IF EXISTS ...`）。
2. 创建 V2 表结构（含 `stock_valuation_daily`、`stock_financial_report`、`sync_job_run`）。
3. 写入初始化数据（管理员用户示例）。
4. 提供导入模板语句（`INSERT ... ON DUPLICATE KEY UPDATE`）。

> 说明：`stock_financial` 已拆分为 `stock_valuation_daily`（日级）与 `stock_financial_report`（报告期级），用于解决“当日筛选”和“跨期变化分析”口径冲突。
