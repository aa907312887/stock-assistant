# 项目启动说明

按以下顺序操作即可在本机跑起 Demo 并看到效果。

---

## 零、一键同时启动前后端（推荐）

在**项目根目录**执行（需已配置数据库与 `.env`，且已执行过 `cd backend && python3 -m venv .venv && .venv/bin/pip install -r requirements.txt` 以及 `cd frontend && npm install`）：

```bash
./scripts/start-dev.sh
```

- 后端：<http://localhost:8000>（后台运行）
- 前端：<http://localhost:5173>（当前终端显示前端日志）
- 按 **Ctrl+C** 会同时停止后端和前端。

**后端日志**：同时输出到**控制台**（运行 `start-dev.sh` 的终端）和**日志文件** `backend/logs/app.log`。出现 500 或“无日志”时，可查看该文件或执行 `tail -f backend/logs/app.log`。

---

## 一、前置：数据库（若尚未准备）

1. 建库建表：
   ```bash
   mysql -u root -p < scripts/init.sql
   ```
2. 插入演示用户（杨佳兴、王悦，密码均为 123456）：
   ```bash
   mysql -u root -p stock_assistant < scripts/insert_users.sql
   ```
3. 在 `backend` 目录下配置 `.env`（可复制 `.env.example`），填写你的 MySQL 连接：
   ```
   DATABASE_URL=mysql+pymysql://用户名:密码@localhost:3306/stock_assistant
   ```

---

## 二、启动后端

**新开一个终端**，在项目根目录执行：

```bash
cd backend
.venv/bin/uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

若尚未创建虚拟环境或安装依赖，先执行：

```bash
cd backend
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

看到类似输出即表示后端已启动：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

- 接口文档：<http://127.0.0.1:8000/docs>
- 健康检查：<http://127.0.0.1:8000/health>

---

## 三、启动前端

**再开一个终端**，在项目根目录执行：

```bash
cd frontend
npm run dev
```

若未安装过依赖，先执行：

```bash
cd frontend
npm install
npm run dev
```

看到类似输出即表示前端已启动：

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

---

## 四、看 Demo 效果

1. 浏览器打开：**http://localhost:5173**
2. 会进入**登录页**（深色渐变背景、居中卡片）：
   - 输入用户名：`杨佳兴` 或 `王悦`
   - 输入密码：`123456`
   - 点击「登录」
3. 登录成功后会跳转到**首页**：
   - 顶部栏：左侧「股票分析助手」、右侧当前用户名和「退出」
   - 中间卡片：欢迎语 + 后续将开放的功能（综合选股、持仓助手、数据与定时任务）
4. 点击「退出」会清除登录态并回到登录页；未登录访问首页也会被重定向到登录页。

---

## 五、一键命令汇总（两个终端）

| 终端一（后端） | 终端二（前端） |
|----------------|----------------|
| `cd backend && .venv/bin/uvicorn app.main:app --reload --host 0.0.0.0 --port 8000` | `cd frontend && npm run dev` |

确保 MySQL 已启动、`stock_assistant` 库和 `user` 表已建好并插入过用户，且 `backend/.env` 中 `DATABASE_URL` 正确，即可按上表启动并访问 http://localhost:5173 看 Demo。
